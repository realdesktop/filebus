Иерархия объектов.

Иерархия объектов RBus строится runtime, а не compile-time.

Каждый объект может иметь:
 - набор параметров (свойств) 
 - набор типизированных дочерних объектов
 - очередь событий


/ <- корневой объект
 - events <- очередь событий корневого объект
 - debuglevel <- свойство корневого объекта

 - iface <- набор дочерних объектов типа iface

   - wlan0 <- дочерний объект типа iface с именем wlan0
   
     - events <- очередь событий интерфейса
     - state <- свойство объекта интерфейса
